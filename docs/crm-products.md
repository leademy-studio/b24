### 1776 — SEO-аудит сайта

1. Технический аудит (Technical Audit)

 Исполнитель проводит проверку программной части сайта:

	-

 1.1. Анализ индексации: проверка файлов robots.txt, sitemap.xml, поиск скрытых от индексации страниц.

	-

 1.2. Техническая диагностика: поиск битых ссылок (404), циклических редиректов, дублей страниц.

	-

 1.3. Анализ скорости (Core Web Vitals): замер скорости загрузки и отзывчивости интерфейса.

2. Аудит качества работы текущего/прошлого подрядчика (SEO-надзор)

 Исполнитель проводит проверку фактического выполнения работ, заявленных в отчетах третьих лиц:

	-

 2.1. Анализ соответствия утвержденной стратегии (дорожной карты) реально выполненным работам за отчетный период.

	-

 2.2. Проверка внедрений: Техническая верификация изменений на сайте (проверка кода, мета-тегов, текстов) на предмет их физического наличия и корректности.

	-

 2.3. Аудит ссылочного бюджета: Проверка целевого расходования средств на закупку ссылок (сравнение рыночной стоимости размещения с заявленной в отчетах).

	-

 2.4. Проверка достоверности KPI: Сверка данных в отчетах подрядчика с прямыми данными систем аналитики (Яндекс.Метрика/Google Analytics) для выявления подтасовки результатов (например, выдача брендового трафика за SEO-результат).

3. Внутренняя оптимизация (On-Page)

 Исполнитель анализирует контент и структуру посадочных страниц:

	-

 3.1. Аудит мета-тегов (Title, Description, H1) на соответствие семантическому ядру.

	-

 3.2. Проверка текстов на уникальность, отсутствие переспама и смысловую ценность.

	-

 3.3. Анализ внутренней перелинковки и распределения ссылочного веса.

4. Юзабилити и коммерческие факторы (UX/UI)

 Исполнитель оценивает удобство сайта для совершения целевого действия:

	-

 4.1. Анализ коммерческих элементов: корзина, формы заявки, контакты, цены.

	-

 4.2. Анализ поведенческих факторов: глубина просмотра, время на сайте, показатели отказов.

5. Внешняя оптимизация (Off-Page)

 Исполнитель анализирует ссылочный профиль сайта:

	-

 5.1. Оценка качества доноров (сайтов, ссылающихся на акцептор).

	-

 5.2. Проверка анкор-листа на безопасность (риск фильтров за переспам).

	-

 5.3. Выявление токсичных ссылок, требующих удаления.
