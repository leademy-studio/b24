# Типовая структура описаний задач — Академия Спорта (GROUP_ID 29)

Документ фиксирует **шаблоны полей в описании** задач и подзадач в разбивке по направлениям.
Используется для проектирования архитектуры БП (автогенерация задач, шаблонизация).
Источник: выборка карточек `tasks.task.get` по группе 29, портал leademy.bitrix24.ru.

## Общие соглашения

- Описания размечены **BBCode** Bitrix24: `[b]…[/b]`, `[url]…[/url]` / `[url=…]…[/url]`, `[list]…[/list]` и `[list=1]` (нумерованный), `[*]` (пункт), `[spoiler]…[/spoiler]`, `[code]…[/code]`, `[quote]`, `[p]`, `[disk file id=…]`.
- Старые задачи (до ~2025) встречаются с HTML-тегами `<b></b>` вместо `[b][/b]` — семантика та же.
- Структура везде одинаковая: **жирная метка-поле + значение под ней**. Ниже метки приведены без BBCode.
- Сквозные служебные поля почти в каждой задаче: **Доступы** (`В группе` / `В папке проекта`) и **ВАЖНО/Важно** (условие завершения задачи).

|------|-----------|-----------------|
| Задача | Услуга месяца | Сопровождение проекта по услуге SEO-продвижения / PPC-продвижения / технической поддержке |

| Регионы | Гео | Тюмень |

## 2. SEO-продвижение — подзадачи

Три подтипа в зависимости от характера работ.

### 2.1. Процедурная подзадача с чек-листом (создание страниц, перелинковка)
| Поле | Назначение |
|------|-----------|
| Задача | Что сделать (+ ссылка на файл услуг/семантики) |
| Порядок работ | Пошаговый список (семантика → ТЗ копирайтеру → контент → согласование → создание → sitemap → индексация GSC/Вебмастер → topvisor) |
| Доступы и файлы | Где материалы (`В папке` / `В группе`) |

**Пример (ID 5677, 5311 «Создание страниц»; 5625 «Оптимизация перелинковки»):**
```
[b]Задача:[/b] Создать 3 страницы, посвященные направлениям из списка [url=…]услуг[/url].
[b]Порядок работ:[/b]
Проверить семантику по группе запросов…
Составить ТЗ копирайтеру… Создать страницу… Обновить sitemap.xml…
Отправить на индексацию в Google Search Console и Яндекс Вебмастер…
[b]Доступы и файлы:[/b] В папке
```

### 2.2. Подзадача с ТЗ-ссылкой (каноникал, микроразметка, создание ТЗ)
| Поле | Назначение |
|------|-----------|
| Задача | Что сделать + `[url]` на ТЗ в Google Docs |
| Доступы | `В группе` |

**Пример (ID 5169 «Настройка тегов каноникал», 5383 «Создание ТЗ для текста»).**

### 2.3. Подзадача-результат (закупка ссылок, ежемесячный отчёт)
| Поле | Назначение |
|------|-----------|
| Задача | Купить ссылки согласно бюджету / Сформировать отчет за период <Месяц Год> |
| Выполненные работы | (заполняется при отчёте) |
| ВАЖНО / Важно | Условие завершения: задачу закрывать только с комментарием-результатом (финальные URL backlink'ов / отчёт) |

**Пример (ID 5821 «Закупка ссылок», 5203 «Формирование ежемесячного отчета»):**
```
[b]Задача:[/b] Купить ссылки для проекта, согласно бюджету
[b]ВАЖНО:[/b] Завершать только когда прописан комментарий с финальными URL backlink'ов и прикреплён как результат
```

## 3. PPC-сопровождение / Контекстная реклама — подзадачи

### 3.1. Запуск рекламной активности (Яндекс Директ / VK / Google Ads)
Самый объёмный шаблон. Шапка-параметры + регламент + развёрнутые чек-листы в спойлерах.

| Поле | Назначение |
|------|-----------|
| Задача | Настроить новую активность по проекту |
| Направление | Тип активности (догоняющая/ретаргетинг, новая РК) |
| Площадки | Яндекс Директ / ВК / Google Ads |
| База | Аудитория (опц., «во вложении») |
| Офферы | Список акций (опц., `[list=1]`) |
| Посадочная страница | URL/статус (опц.) |
| Регламент запуска рекламных кампаний | Нумерованный список: настроить → скриншоты на согласование → запуск после согласования |
| География | Тюмень |
| Бюджет | напр. 10 000 |
| Период | напр. 14 дней / До 30 мая |
| Доступы | `В группе` |
| Файлы | `Во вложении` |
| Чек-лист запуска РК … | `[spoiler]` блоки: ЕПК Поиск, ЕПК РСЯ, ЕПК «Конверсии и трафик», Google Ads — семантика, кластеризация, объявления, креативы, целевой URL, доп. ссылки, уточнения, промоакции, UTM-разметка (`[code]`), согласование, запуск |

**Пример (ID 5789 «VK», 5635 «Яндекс Директ»).** Чек-листы в спойлерах — стандартные, переносятся из месяца в месяц без изменений.

### 3.2. Технические подзадачи PPC (антибот, ROISTAT, счётчики)
| Поле | Назначение |
|------|-----------|
| Задача | Что настроить |
| Описание из системы / Описание | Тех.инструкция (DNS, IP, шаги) |
| Доступы | Ссылки + логин/пароль сервисов (ELAMA, Killbit и т.п.) |

**Пример (ID 5307 «Установка антибот настроек»).**

### 3.3. Ежемесячный отчёт PPC
Идентичен SEO-отчёту (2.3): поля **Задача** (за период <Месяц Год>) / **Выполненные работы** / **Важно** (завершить с комментарием-отчётом как результат). Пример: ID 5443, 3505.
