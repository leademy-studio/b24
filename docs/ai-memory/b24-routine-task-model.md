Bitrix24 portal https://leademy.bitrix24.ru, working webhook scopes (проверено 2026-06-17, после расширения): bizproc, crm, **im**, im.import, imconnector, notifications, socialnetwork, task, tasks, tasks_extended, user, userfieldconfig. Scope `im` выдан → постинг в чаты (im.message.add) и im.dialog.get доступны (конспекты внутр. планёрок в чаты, уведомления владельцу). (env var B24_WEBHOOK_BASE in .env; webhook code d2aegrf5yyy9gw4z as of 2026-06-10.) Read-only сверка портала — scripts/verify-data.mjs (node --env-file=.env).

Business model = monthly subscription per service. Each service in a project (workgroup) becomes one global monthly parent task `<Project> — <Service> (<Month Year>)` (PARENT_ID 0/null) plus typed routine subtasks. Services: SEO-продвижение, PPC-сопровождение (контекстная реклама), Тех.поддержка, Администрирование, Программирование, Разработка дизайн-макета.

SEO package (CRM products 87/1805/1801 = S/M/L) drives quantity: N pages = 3/5/10/month → N×"Создание страниц", N×"Создание ТЗ"; plus 1× each of Сбор семантического ядра, Оптимизация перелинковки, Закупка ссылок, Ежемесячный отчёт. PPC = "Запуск рекламной активности" per platform + Ежемесячный отчёт. WEEKLY PPC task (template #10, example task 1877) "Оптимизация рекламы в Яндекс Директ": created every Wednesday, deadline Friday 19:00 (= start +2 days, no RF calendar), responsible 103 +accomplice 17, under current month PPC parent — needs a separate weekly Cloud Scheduler (cron/launch-weekly), not the monthly run. Routine subtask exact texts (templates #1–9, incl. #9 ad-hoc "обновление/запуск РК по запросу с созвона" which is NOT auto-scheduled) are in docs/rutinnye-podzadachi-soderzhimoe.md; service tree in docs/derevo-uslug-rutinnye-zadachi.md.

SOURCE OF TRUTH for what to generate = recurring deals in the "Производство" deal pipeline (CRM category ID 0, isDefault). crm.deal.recurring.list = 9 active (live 2026-06-17; было ~12 на 2026-06-10), monthly, NEXT_EXECUTION 2026-07-01. Воронки сделок (entityTypeId=2): 0 Производство(default), 3 Квалификация, 5 Первая продажа лицензии, 7 Продажа продления, 9 Продажа услуг — продажи идут в 3/5/7/9, производство в 0. Лиды включены (~96). Денис Сафонов = user 31. Each base deal carries: UF_CRM_1727554217 = project GROUP_ID (e.g. Phoenix deal 97 → group 9; Трубопровод72 deal 113 → group 35); UF_CRM_1725984512097 = services multi-enum [139=SEO,141=PPC,143=SMM,145=Контент,147=Аренда сайта,149=Поддержка,151=Разработка,287=Комплекс,293=Пополнение балансов]; UF_CRM_1727619728 = site; UF_CRM_1726088408701 = project name. SEO package S/M/L comes from the deal's product row: PRODUCT_ID 87→S(3 pages), 1805→M(5), 1801→L(10); default S if no/other product. Responsibles (fixed rules, user scope now granted): SEO parent+all subtasks → 101 Максим Логвинов; PPC parent → 17 Святослав Павлишин; PPC subtasks → responsible 103 Владислав Павлишин + accomplice (ACCOMPLICES) 17. So the external service is STATELESS — no own datastore, reads everything from CRM/Bitrix. Current Bitrix automation = robot on deal (creates global task, Layer A) + robots in tasks (spawn subtasks, copied manually, looping BPs, Layer B); both disabled at cutover, recurring deals kept.

Useful: list groups = socialnetwork.api.workgroup.list (53 groups); tasks per group = tasks.task.list filter GROUP_ID; products = crm.product.list (32); deal pipelines = crm.category.list entityTypeId=2. user.get blocked (no `user` scope) — only IDs (SEO→101, PPC→17, creator→1).
