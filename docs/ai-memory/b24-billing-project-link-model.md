Модель биллинга и связи «проект↔клиент» для портала `leademy.bitrix24.ru`. Подробности — в [docs/derevo-uslug-rutinnye-zadachi.md](../../docs/derevo-uslug-rutinnye-zadachi.md) §8–9. Связано с [[b24-routine-task-model]], [[owner-dashboard-project]].

**Биллинг — предоплата:** счёт за месяц выставляется заранее, работа после оплаты → оплата гейтит генерацию пакета задач.

**Счета = смарт-счёт** `entityTypeId=31` (классические `crm.invoice` НЕ используются). «Оплачено» ⇔ `stageId=DT31_9:P` (`SEMANTICS=S`, «Оплачен»); стадии: N Новый / S Отправлен / P Оплачен / D Не оплачен (`ENTITY_ID=SMART_INVOICE_STAGE_9`). Поля: `closedate`=срок оплаты, `opportunity`=сумма, `parentId2`=сделка, `companyId`. Период в названии НЕ кодируется (там клиентские коды).

**Сделки-производство:** воронка `CATEGORY_ID=0`, стадии `NEW`(Получение оплаты)/`EXECUTING`(В работе)/`WON`/`LOSE`. Продления — цепочка через поле `UF_CRM_PREVIOUS_DEAL`.

**Проект = рабочая группа** (`sonet_group`, нужен scope `sonet_group`). Мост группа→сделка — через задачи: `UF_CRM_TASK=["D_<dealId>"]`. Клиент может иметь **несколько юрлиц** → проект(группа) 1:N компаний; ключ связи = `UF_CRM_GROUP_ID` (integer) НА КОМПАНИИ; активность проекта = оплата любого из юрлиц.

**КРИТИЧНО — дисциплина статусов мертва (на 2026-06):** последний счёт в «Оплачен» 25.01.2026; группы все ACTIVE, CLOSED=0; сделок в EXECUTING всего 1.

**Порядок реализации (решение владельца 2026-06-19):** (1) ✅ структурная связь группа↔компания `UF_CRM_GROUP_ID` — сделано; (2) мониторинг-слой на дашборде; (3) генератор рутинных задач, гейт оплаты в ручном/мягком режиме; (4) **T-API токен + робот оплаты — ПОСЛЕДНИЙ приоритет** (доступы `bank-statements/get`+`bank-accounts/get`). Пока (4) не сделан — жёсткий автогейт и «активность по факту оплаты» работают вручную/полуавтоматически. Поле формы выпуска токена «адрес» = статический IP сервера робота (его ещё нет; можно оставить без IP-ограничения для старта).

Связь group↔company реально M:N (юрлицо 123 = 4 группы) → ключ `UF_CRM_GROUP_ID` сделан МНОЖЕСТВЕННЫМ. Среди 54 групп есть внутренние/тестовые (LEADEMY, тестовый проект) — не клиентские, исключать из списка проектов.

**Создание компании:** обязательное поле `UF_CRM_1723378663476` (тип юрлица): `45=ООО`, `47=ИП`, `49=СЗ`. Реквизиты — пресет 1 «Организация» (crm.requisite.add) + crm.requisite.bankdetail.add.

**Созданные поля (портал leademy):** `UF_CRM_CONV_DATA` (сделка, enum Да/Нет/Не выяснено, ID 1407, гейт задачи feedback-сигналов); `UF_CRM_CASH_PAYER` (компания, boolean, ID 1409, наличники — обход банк-детекта); `UF_CRM_GROUP_ID` (компания, integer МНОЖЕСТВЕННОЕ, ID 1413, связь юрлицо→проект(группа); бутстрап на 2026-06-19 проставил 19 компаний→22 группы).
