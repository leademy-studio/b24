User is the business owner of a marketing agency (portal https://leademy.bitrix24.ru). Goal: build their own web dashboard to monitor agency projects and the active routine tasks per service (SEO / PPC) and per project, and to perform actions (deploy a month's tasks by service/package, create ad-hoc campaign tasks).

Decisions made (2026-06-10):
- Scope: monitoring + actions (not read-only).
- Approach: write plan/ТЗ first, no code yet. ТЗ lives at docs/dashboard-tz.md.
- Hosting: GCP — Cloud Run (Node.js backend extending server.js) + Firebase Hosting (static UI) + Secret Manager for the Bitrix webhook.
- Independence level 1: own UI/logic on top of Bitrix; Bitrix stays the system of record (no own datastore yet).

Added requirement (2026-06-10): from **July 2026** this external service also becomes the **scheduler** that auto-creates monthly routine tasks for active projects (Cloud Scheduler → Cloud Run /api/cron/launch-month, idempotent, Firestore subscription registry). Cutover = 1 July 2026. CRITICAL: an existing Bitrix recurring-task template/robot currently creates the monthly parents (1st of month ~00:03, creator user 1 = Равиль Шакиров the owner); it MUST be disabled before the external scheduler is enabled or projects get duplicate tasks. July 2026 parents do not exist yet as of 2026-06-10 (clean cutover window). Tight timeline (~3 weeks); fallback = keep Bitrix automation for July, cutover 1 Aug.

Also in scope (2026-06-10): dashboard shows finances (budgets/prices/deal sums). Separate T-Bank integration: on incoming payment matched by payer INN, auto-move the deal from stage NEW ("Получение оплаты от клиента") to EXECUTING ("В работе") in pipeline cat 0. Match path: parse invoice number from payment purpose, format = {CLIENT_ID=COMPANY_ID}-{SELF_ID=DEAL_ID}-{MM}-{YYYY} (confirmed via 115-1119-06-2026 → company 115 РЕЛАКС, deal 1119) → crm.deal.get(SELF_ID) verify CATEGORY_ID=0 & STAGE_ID=NEW & COMPANY_ID==CLIENT_ID & payer INN==company RQ_INN → crm.deal.update STAGE_ID=EXECUTING. Fallback if unparsable: payer INN → company → single NEW deal in cat 0. Invoices = smart process entityTypeId 31, linked to deal via parentId2; printed number NOT stored in any field. Exclude own INN 7203566588 (ООО ЛИДЕМИ). T-Bank = T-Business OpenAPI (base https://business.tbank.ru/openapi/api/, Bearer token in Secret Manager), events via webhook on incoming credit (+ statement polling as backup). Partial payment (amount < deal OPPORTUNITY) → do NOT move stage, only on full payment. Idempotency by operationId. Subcontract projects: boolean deal field UF_CRM_SUBCONTRACT (id 1403, created 2026-06-10) — such deals auto-move NEW→EXECUTING WITHOUT payment (T-Bank matcher skips them); transition done ONLY at month-start launch (no periodic check). Responsibles finalized: SEO & Администрирование(149)→101 Максим Логвинов; PPC parent→17 Святослав Павлишин; PPC subtasks→103 Владислав Павлишин +accomplice 17. Global auto-task deadline = +1 month +3 RF working days (isdayoff.ru calendar). Service 149 "Поддержка"→ only a parent task "<Project> — Администрирование сайта (Месяц Год)", no subtasks; only 139/141/149 generate tasks.
