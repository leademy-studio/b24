# Образцы для ингеста конспектов

Тестовые образцы для парсера входящих писем Telemost (Этап 4, см. [../../avtomatizatsiya-konspektov-plan.md](../../avtomatizatsiya-konspektov-plan.md) §7).

## `telemost-email-sample.eml`

**Обезличенный** фикстур: точно повторяет **структуру** реального письма Яндекс Телемост (проверено на письме от 12.06.2026), но содержимое разговора — **синтетическое** (без клиентских данных). Реальные письма в git не коммитим (см. `.gitignore`).

Что воспроизводит (на этом парсер ингеста тестируется как на боевом письме):
- отправитель `Хранитель встреч <…>`, тема `Конспект встречи`;
- структура `multipart/mixed` → `multipart/related` (HTML-тело = саммари Яндекса + встроенная `cid:`-картинка) + **вложение**;
- вложение: `text/plain`, `Content-Transfer-Encoding: base64`, `Content-Disposition: attachment`, имя `ГГГГ-ММ-ДД ЧЧ:ММ (MSK) <meetingId>.txt`;
- содержимое вложения = сырая расшифровка, начинается с `Встреча проходила ДД.ММ.ГГГГ с ЧЧ:ММ (MSK)…` (есть «вода» — приветствия, отвлечения — для проверки очистки на Этапе 1).

Парсер должен: взять часть с `Content-Disposition: attachment` и `text/plain`, раскодировать base64 → UTF-8, извлечь `meetingId` из имени файла (ключ дедупа), сохранить сырой текст. Фоллбэк — тело письма, если вложения нет.
