Всего задач в группе: **178**. Источник: `tasks.task.get` / `tasks.task.list`, портал leademy.bitrix24.ru.

## Паттерн бизнес-процесса (как устроены задачи)

На каждый календарный месяц по каждой услуге заводится **одна глобальная родительская задача** (`PARENT_ID = 0/null`), а конкретные работы оформляются **подзадачами** внутри неё.

- **Именование родителя:** `Академия Спорта — <Услуга> (<Месяц Год>)`
- **Услуги (направления):** `SEO-продвижение`, `PPC-сопровождение` (= контекстная реклама; в 2024 называлось `Контекстная реклама`), плюс эпизодические `Администрирование сайта`, `Программирование сайта`, `Тех.поддержка сайта`, `Разработка дизайн-макета`.
- **Жизненный цикл месяца:** создаётся 1-го числа (`CREATED_DATE`), дедлайн = 1-е число следующего месяца (`DEADLINE`), статус идёт `Ждет выполнения (2)` → `Завершена (5)`.
- **Типовые подзадачи SEO:** «Закупка ссылок», «Создание страницы», «Сбор семантического ядра», «Микроразметка», «Оптимизация перелинковки», «Формирование ежемесячного отчета».
- **Типовые подзадачи PPC/контекст:** «Запуск рекламной активности: Яндекс Директ / VK», «Обновление РК», «Чистка площадок», «Антибот», «Подключение ROISTAT», «Формирование ежемесячного отчета».
- **Описание родителя** — единый шаблон с полями: Задача, Сайт, Регионы, Рекламный бюджет, Доступы.
