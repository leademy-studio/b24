# Точное содержимое рутинных подзадач — Академия Спорта (GROUP_ID 29)

Каждая рутинная подзадача из месяца в месяц создаётся с **идентичным описанием** (проверено сравнением экземпляров за разные месяцы). Ниже — дословное содержимое (BBCode как в Bitrix24) каждого типа. Переменные части помечены как `{{…}}`; всё остальное — фиксированный boilerplate, переносимый без изменений.

> Замечание по разметке: часть задач хранится с HTML-тегами `<b></b>`, часть — с BBCode `[b][/b]`. Семантика идентична; ниже приведено как в исходных карточках.

### 9. Обновление / запуск рекламной активности по запросу клиента (внеплановая)
_ID-примеры: 1561 «Обновление РК», 2585 «Обновление рекламных объявлений», 4215 «Обновление рекламной активности (Black Friday)», 831 «Запуск новой активности в Я.Директ», 3197 «Анализ работы РК»._

**Природа задачи:** возникает по итогам коммуникации с клиентом (еженедельные/периодические созвоны) из-за **динамической смены маркетинговых активностей** — новая акция, смена оффера/цены, обновление креативов и баннеров, сезонная распродажа. **Запланировать заранее нельзя** — создаётся по триггеру «договорённость на созвоне». Это рутинный *тип* задачи (повторяется регулярно), но не рутинный *по расписанию*.

Шаблон гибкий: используется **облегчённая шапка** (обновление оффера/цены/креатива) либо **полная** (запуск новой РК — тогда добавляется блок «Регламент» + чек-листы из шаблона №7). Переменно почти всё, кроме структуры полей и регламента/чек-листов.

```
[B]Задача:[/B]
{{Что сделать: обновить активность / обновить объявления и креативы / запустить новую активность}}
{{Повод: новая акция / смена оффера / сезонная распродажа (напр. Black Friday) / снижение стоимости заявки}}

[B]Повод (источник):[/B]
{{Договорённость на созвоне с клиентом от {{дата}} / письмо / чат}}

[B]Направление:[/B]
{{напр. Black Friday / Догоняющая реклама / Новый оффер «фитнес с бассейном»}}

[B]Площадки:[/B]
{{Яндекс Директ / Google Ads / ВК — все или выбранные кампании (РСЯ, МК и т.д.)}}

[B]Что изменить (было → стало):[/B]
{{Старый оффер/цена: «… 5 659 ₽» → Новый: «… 5 093 ₽».
Где встречается: заголовки, тексты, быстрые ссылки, креативы. Заменить креатив со старой ценой.}}

[B]Условия / Офферы:[/B]
{{Текст актуальной акции и условия (что входит, цена, выгода)}}

[B]Период:[/B]
{{Даты акции: с {{дд.мм}} до {{дд.мм}}; что приоритетно (напр. цену поменять сегодня, тексты — завтра)}}

[B]География:[/B]
{{Тюмень}}

[B]Бюджет:[/B]
{{при необходимости — сумма на период}}

[B]Материалы:[/B]
{{[URL=…]Промо-материалы / новые креативы[/URL] / Во вложении}}

[B]Доступы:[/B]
В группе

[B](опционально, если это ЗАПУСК новой РК, а не правка)[/B]
[B]Регламент запуска рекламных кампаний[/B]
[list=1]
[*]Сформировать и настроить рекламные кампании по чеклисту[*]Предоставить скриншоты объявлений (включая расширения) на согласование[*]Запускать рекламные кампании после согласования
[/list]
+ блок «Чек-лист запуска РК…» из шаблона №7 (Яндекс Директ ЕПК Поиск / РСЯ / Конверсии-трафик / Google Ads) — переносится без изменений.
```

> Подвид «Анализ работы РК» (ID 3197) — лёгкая аналитическая версия без шапки: свободный текст «посмотреть статистику, предложить, за счёт чего снизить стоимость заявки; мысли в комментарии, при необходимости — созвон».

## Итог для шаблонизатора

Рутинные подзадачи = **константные «рыбы»** с минимальным числом плейсхолдеров:

| Подзадача | Направление | Переменные части |
|-----------|-------------|------------------|
| Закупка ссылок | SEO | — (полностью статична) |
| Создание страниц | SEO | кол-во `{{N}}`, ссылка на файл услуг |
| Сбор семантического ядра | SEO | направление, регион |
| Создание ТЗ для текста | SEO | предмет ТЗ |
| Оптимизация перелинковки | SEO | — (полностью статична) |
| Ежемесячный отчёт SEO | SEO | `{{Месяц Год}}` |
| Запуск РК: Яндекс Директ/VK | PPC | шапка (направление/площадки/база/офферы/бюджет/период); чек-листы статичны |
| Ежемесячный отчёт PPC | PPC | `{{Месяц Год}}` |
| **Обновление/запуск РК по запросу (внеплановая)** | PPC | **по триггеру с созвона**; вся шапка переменна, регламент/чек-листы статичны |

При автогенерации через `tasks.task.add` достаточно хранить эти тексты как константы и подставлять плейсхолдеры.

> Особый случай — внеплановая задача №9. Её **нельзя положить в авто-расписание месяца** (зависит от динамики маркетинга клиента). В БП она заводится вручную/полуавтоматически по триггеру «договорённость на созвоне», но использует тот же шаблон-«рыбу». Рекомендация: создавать через быстрый шаблон/кнопку в карточке проекта, а не через ежемесячный генератор.
