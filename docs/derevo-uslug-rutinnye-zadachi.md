# Древо услуг и рутинные задачи: SEO и PPC

Бизнес-анализ связки **«услуга из CRM-каталога → набор рутинных подзадач в проекте»** для проектирования архитектуры БП (автогенерация месячного плана работ).

Источники: [crm-products.md](crm-products.md) (каталог услуг), [rutinnye-podzadachi-soderzhimoe.md](rutinnye-podzadachi-soderzhimoe.md) (эталоны подзадач), [akademia-sporta-task-examples.md](akademia-sporta-task-examples.md) (структура месячного цикла), [struktura-opisaniy-zadach.md](struktura-opisaniy-zadach.md).

---

## 1. Главный вывод

Бизнес-модель агентства = **подписка на ежемесячное сопровождение по услуге**. Каждая услуга в CRM (сделка/товар) разворачивается в проекте Bitrix24 в **одну глобальную месячную задачу-родитель** + **типовой набор рутинных подзадач**. Состав и количество подзадач **детерминированы услугой и её пакетом** — значит, генерацию месячного плана можно полностью шаблонизировать.

Ключевая находка: **пункты в описании товара 1:1 соответствуют рутинным подзадачам.** Описание SEO-пакета (товар 87) — это, по сути, чек-лист подзадач месяца:

| Блок в описании услуги (товар 87) | Рутинная подзадача |
|-----------------------------------|--------------------|
| Контентная оптимизация → Сбор семантики | **Сбор семантического ядра** |
| Контентная оптимизация → Подготовка текстов | **Создание ТЗ для написания текста** |
| Контентная оптимизация → Публикация контента на N страницах | **Создание страниц** (×N по пакету) |
| Контентная оптимизация → Внутренняя перелинковка | **Оптимизация перелинковки** |
| Усиление ссылочной массы (off-page) → Размещение ссылок | **Закупка ссылок** |
| Аналитика и отчётность → Подготовка отчёта | **Формирование ежемесячного отчёта (SEO)** |
| Техническая оптимизация и контроль | (входит в сопровождение родителя; точечно — тех. подзадачи) |

То же для PPC: «Настройка и запуск РК», «аналитика», «масштабирование/отключение связок» → подзадачи **Запуск рекламной активности** + **Ежемесячный отчёт** + точечные тех. задачи.

## 3. Древо услуг → рутинные задачи

```
АГЕНТСКИЕ УСЛУГИ (подписка, помесячно)
│
├── 🟦 SEO-ПРОДВИЖЕНИЕ
│   │   CRM: 87 / 1805 / 1801 (пакеты S/M/L), 499
│   │   Родитель месяца: «<Проект> — SEO-продвижение (<Месяц Год>)»
│   │   Параметры родителя: Сайт, Регион, Рекламный бюджет, Доступы
│   │
│   ├── РУТИННЫЕ ПОДЗАДАЧИ (каждый месяц):
│   │   ├── Сбор семантического ядра ............... 1× (под план страниц)         [статична + регион/направление]
│   │   ├── Создание ТЗ для написания текста ....... = N страниц пакета            [предмет ТЗ]
│   │   ├── Создание страниц ....................... N = 3 (S) / 5 (M) / 10 (L)     [N, ссылка на файл услуг]
│   │   ├── Оптимизация перелинковки ............... 1×                            [полностью статична]
│   │   ├── Закупка ссылок ......................... 1× (в рамках бюджета)         [полностью статична]
│   │   └── Формирование ежемесячного отчёта (SEO) . 1× (закрытие месяца)          [Месяц Год]
│   │
│   └── РАЗОВЫЕ / ПО ТРИГГЕРУ (не каждый месяц):
│       ├── SEO-аудит сайта (CRM 1776) ............. при старте проекта
│       ├── Технические правки (микроразметка, каноникал, alt, H-теги, sitemap)
│       └── Тех. оптимизация / правки на сайте (передаётся в Программирование/Админ.)
│
└── 🟥 PPC-ПРОДВИЖЕНИЕ / КОНТЕКСТНАЯ РЕКЛАМА
    │   CRM: 477 / 1809
    │   Родитель месяца: «<Проект> — PPC-сопровождение (<Месяц Год>)»
    │   Параметры родителя: Сайт, Регион, Рекламный бюджет, Доступы
    │
    ├── РУТИННЫЕ ПОДЗАДАЧИ (каждый месяц):
    │   ├── Запуск рекламной активности ............ 1×+ на площадку/направление    [шапка перем.; чек-листы статичны]
    │   │     ├─ Яндекс Директ (ЕПК: Поиск / РСЯ / Конверсии-трафик)
    │   │     ├─ Google Ads (Поиск / Дисплей)
    │   │     └─ VK (для таргет/догоняющей)
    │   └── Формирование ежемесячного отчёта (PPC) .. 1× (закрытие месяца)          [Месяц Год]
    │
    ├── РЕГУЛЯРНЫЕ, НО ВНЕПЛАНОВЫЕ (по триггеру с созвона, см. шаблон №9):
    │   └── Обновление / запуск РК по запросу .......... частота не фиксирована
    │         ├─ обновить баннеры / креативы
    │         ├─ сменить оффер / цену в объявлениях
    │         ├─ запустить новую акцию (сезонная, Black Friday)
    │         └─ анализ работы РК / оптимизация стоимости заявки
    │
    └── РАЗОВЫЕ / ПО ТРИГГЕРУ:
        ├── Пополнение баланса (CRM 425 / 1813) ..... по необходимости (биллинг)
        ├── Установка антибот-настроек (DNS-экран, Killbit) . при защите от скликивания
        ├── Подключение ROISTAT / счётчиков / Метрики / GTM .. при старте/донастройке
        └── Чистка площадок / обновление РК / анализ работы РК
```

> **Класс «регулярные, но не планируемые».** Задачи обновления/запуска РK по запросу — это рутинный *тип* (повторяется почти каждый месяц), но **не рутина по расписанию**: повод и содержание диктуются динамикой маркетинговых активностей заказчика и фиксируются на еженедельных/периодических созвонах. В БП они НЕ попадают в ежемесячный авто-генератор, а создаются по триггеру «договорённость на созвоне» из готового шаблона (см. шаблон №9 в [rutinnye-podzadachi-soderzhimoe.md](rutinnye-podzadachi-soderzhimoe.md)). Рекомендуемая реализация — кнопка/быстрый шаблон в карточке проекта.

## 4. Правила привязки (для шаблонизатора БП)

При заведении услуги в проекте генерируется месячный план по правилам:

**SEO (пакет → количество):**
- `Создание страниц`: N = {S→3, M→5, L→10} (берётся из выбранного товара-пакета).
- `Создание ТЗ для написания текста`: по числу страниц, требующих копирайта (обычно = N или меньше, если часть страниц технические).
- `Сбор семантического ядра`, `Оптимизация перелинковки`, `Закупка ссылок`, `Ежемесячный отчёт`: ровно по 1 шт/мес независимо от пакета.
- Все тексты подзадач — **константы**, подставляются только: `{{N}}`, `{{ссылка на файл услуг}}`, `{{регион}}`, `{{Месяц Год}}`.

**PPC:**
- `Запуск рекламной активности`: 1 задача на каждое согласованное направление/площадку (Яндекс Директ / Google Ads / VK). Чек-листы в спойлерах — константы; переменна только шапка (Направление, Площадки, Бюджет, Период, Офферы).
- `Ежемесячный отчёт (PPC)`: 1 шт/мес.
- Мультирегион (товар 1809) → запуск/ведение РК масштабируется на число регионов.

**Общие параметры родителя месяца** (оба направления): `Сайт`, `Регион`, `Рекламный бюджет (Месяц Год)`, `Доступы` — наследуются из карточки проекта/сделки.

## 6. Наблюдения и рекомендации

1. **Высокая стандартизация → готово к автогенерации.** 6 рутинных подзадач SEO и 2 PPC покрывают основную массу месячной работы. Их можно хранить как шаблоны (`tasks.task.add`) с 3–4 плейсхолдерами и спавнить пачкой при создании сделки/наступлении месяца.
2. **Объём SEO привязан к пакету (S/M/L)** — это единственный реальный «переключатель» количества задач. Стоит вынести `N страниц` в свойство товара/сделки и пробрасывать в генератор.
3. **«Закупка ссылок» и «Перелинковка» — полностью статичны** (0 плейсхолдеров) → идеальные кандидаты на полную автоматизацию без участия менеджера.
4. **«Ежемесячный отчёт»** есть в обоих направлениях, текст идентичен (меняется только период) → один общий шаблон с параметром услуги.
5. **Разделение рутины и триггерных задач.** Тех. правки сайта, антибот, ROISTAT, пополнение баланса — не рутина расписания, а реакция на событие; их не надо генерировать автоматически каждый месяц.
6. **Пробел в данных:** у многих товаров `PRICE = 0` и нет привязки `SECTION_ID` к разделу каталога — каталог не структурирован по направлениям. Для чистой архитектуры стоит сгруппировать товары в разделы «SEO / PPC / Разработка / Поддержка» и проставить цены пакетов.
7. **Несогласованность ТЗ-задач:** число «Создание ТЗ» и «Создание страниц» не всегда совпадает (часть страниц технические/без копирайта). В генераторе развести: `страниц с копирайтом` vs `технических страниц`.

## 7. Предлагаемый сценарий БП (псевдо-флоу)

```
Сделка выиграна (услуга = SEO-пакет L)
        │
        ▼
[Старт проекта] → SEO-аудит (1776, разово) ─┐
        │                                    │
        ▼                                    ▼
[1-е число месяца] ── создать родителя «… — SEO-продвижение (Месяц Год)»
        │                 (Сайт/Регион/Бюджет/Доступы из сделки)
        ▼
   авто-спавн подзадач по пакету L:
   ├─ Сбор семантического ядра ×1
   ├─ Создание ТЗ ×10
   ├─ Создание страниц ×10
   ├─ Оптимизация перелинковки ×1
   ├─ Закупка ссылок ×1
   └─ Ежемесячный отчёт SEO ×1   (дедлайн = 1-е число след. месяца)
        │
        ▼
[Закрытие месяца] отчёт-результат → следующий месяц (повтор цикла)
```
PPC-флоу аналогичен: родитель «… — PPC-сопровождение» → «Запуск рекламной активности» на каждую площадку + «Ежемесячный отчёт PPC».
